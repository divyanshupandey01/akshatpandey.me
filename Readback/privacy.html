Introduction

ReadBack helps users automatically bookmark and restore their reading position on text-heavy web pages. We believe that what you read is your business alone.

Privacy is one of the core principles of ReadBack. Unlike traditional tools, the extension does not require you to create an account, does not upload your browsing logs, and stores everything locally inside your device sandbox environment.

👤

Information We Collect

ReadBack is built with a zero-telemetry architecture. You can install the extension and start reading immediately. We do not prompt you to sign up, sign in, or subscribe.

We Collect 0 Personal Data

ReadBack collects and transmits exactly **none** of the following pieces of credentials:

✕ Name

Reading Progress

To provide its core bookmarking features, the extension records metrics associated with your current reading position.

What is Stored:
  • Scroll coordinates (Y pixel position)
  • Saved page URLs and titles
  • Custom folders, favorites, and notes
  • Daily reading duration stats
  • Chronological session timeline history
Where it is Stored:

All reading parameters are written directly to your browser's local sandbox storage using Chrome extension APIs. They never leave your device.

⚙️

Required Chrome Permissions

Chrome extensions request permissions to interact with browser modules. ReadBack strictly keeps its footprint minimal:

storage
Local Settings & Databases

Used to persist your folder structures, page highlights, notes, configurations, and session timeline history locally.

activeTab
Current tab interaction

Invoked when you click the extension popup or context menu to extract page titles, normalized URLs, and visible paragraph heights.

alarms
Periodic background cleanups

Triggers a background clean loop to periodically purge reading records older than 6 months (excluding favorites) and keep databases slim.

contextMenus
Right-click save triggers

Injects context menu options ("Save Current Page", "Open ReadBack Dashboard") to facilitate rapid bookmarking without opening the popup.

host_permissions
Content Script Matching

Allows content scripts to execute on reading pages to observe scroll changes, generate stable CSS selectors, and apply paragraph highlights.

Third-Party Services

ReadBack utilizes trusted distribution services, primarily the **Google Chrome Web Store**, to host, update, and manage the extension lifecycle.

These distribution platforms may collect anonymous telemetry regarding installs, updates, and browser versions. No page contents, URLs, or scroll indicators are ever sent to these services.

🔒

Security Architecture

We prioritize data isolation and secure code design principles:

  • **Isolated Sandbox**: Local storage matches Chrome sandbox regulations, locking reading statistics strictly to the extension profile.
  • **No External Scripting**: We run exactly zero remote scripting tags (no Google Tag Manager, analytics wrappers, or external scripts) to prevent DOM-injection vulnerabilities.
  • **Zero Network Hooks**: The service worker contains no network connection dependencies. Even if your browser is offline, bookmarks function perfectly.
